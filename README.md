## 01. в—Ҳ What Is This?

The graph's **layout is never manually set**. Nodes repel each other like charged particles, edges pull same-category projects together via spring forces, and the whole system settles into a topology that mirrors the actual structure of the work.

> Pre-baking 150 physics steps before the first frame means the graph appears already-settled on load вҖ” no jarring explosion of nodes.

## 05. в—Ҳ Architecture

```
neural-map/
в”Ӯ
в”ңв”Җв”Җ index.html                      вҶҗ single file В· zero build step
в”Ӯ
в”ңв”Җв”Җ в”Җв”Җ DATA LAYER в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
в”Ӯ   в”ңв”Җв”Җ RAW[]                       project nodes (name, score, gh, demo)
в”Ӯ   в”ңв”Җв”Җ CATS{}                      category config (color, radius, tags)
в”Ӯ   в””в”Җв”Җ classify()                  rule-based category assignment
в”Ӯ
в”ңв”Җв”Җ в”Җв”Җ PHYSICS LAYER в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
в”Ӯ   в”ңв”Җв”Җ runForce(steps)             custom 3D force simulation
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ repulsion               O(nВІ) Coulomb-style node repulsion
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ attraction              Hooke's law on category edges
в”Ӯ   в”Ӯ   в””в”Җв”Җ gravity                 center pull + velocity damping
```

## 06. в—Ҳ Physics Simulation

The layout is computed by a **custom 3D force-directed algorithm** вҖ” no D3, no physics library.

```
  Each simulation tick (pre-baked В· 150 iterations before first render):

  REPULSION    Fбөўвұј  =  900 / |rбөў вҲ’ rвұј|ВІ         nodes push apart
  ATTRACTION   Fбөўвұј  =  (d вҲ’ 28) Г— 0.018          linked nodes pull together
  GRAVITY      F    =  вҲ’0.012 Г— position          global centering force
  DAMPING      v    =  v Г— 0.78                   energy dissipation
```

| Force | Effect | Visual Result |
|:------|:-------|:-------------|
| Repulsion | Nodes spread out | No overlapping nodes |
| Attraction | Same-category nodes cluster | Domain groups emerge naturally |
| Cross-edges | Random inter-category links | Bridge connections reveal skill overlap |
| Complexity score | Controls `SphereGeometry` radius | Heavier projects visually dominant |
| Gravity | Keeps graph centered | Graph stays in view without clamping |

## 13. в—Ҳ Customization

Append to the `RAW` array inside `<script>`:

```js
{

  demo: "https://my-project.vercel.app/"  // null if no live demo
```

### Change category rules

Edit the `classify()` function:

```js
function classify(name, score) {
  if (name.includes('myterm')) return 'ai';
  if (score > 200000)         return 'fullstack';
  // add your own rules...
}
```

### Tune physics constants

```js
const f = 900 / d2;          // repulsion strength  (вҶ‘ = more spread)
const f = (d - 28) * 0.018;  // spring rest-length & stiffness
n.vx *= 0.78;                 // damping  (вҶ“ = bouncier, slower settle)
```

### Reduce particle count (mobile / lower-end GPUs)

```js
const PCOUNT = 80;   // default 140
```

## 14. в—Ҳ Performance

| Metric | Value | Notes |
|:-------|:-----:|:------|
| Render target | `60 FPS` | `requestAnimationFrame` loop |
| Initial force steps | `150` | Pre-baked before first render |
| Background particles | `140` | Reduce `PCOUNT` for lower-end devices |
| Plexus connections | Dynamic | Distance-threshold proximity links |
| Pixel ratio cap | `min(devicePixelRatio, 2)` | Prevents 4K overdraw |
| Fog | `FogExp2 В· 0.0028` | Natural depth cue, free GPU hint |
| Bundle | `~260KB` | Three.js r128 via CDN, zero else |

**Bottlenecks by priority:**

```
  1.  O(nВІ) repulsion   вҶ’  acceptable at n=23 В· use Barnes-Hut for n>100
  2.  Background plexus вҶ’  reduce PCOUNT or disable for mobile
  3.  Glow halos        вҶ’  additive blending is cheap В· safe to keep
  4.  Labels (Sprites)  вҶ’  Canvas texture per node В· regenerate on demand only
```
